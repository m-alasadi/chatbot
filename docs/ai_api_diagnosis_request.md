# طلب دراسة وتشخيص آلية إجابة الذكاء الاصطناعي عبر الـ API

أريد منك دراسة المشروع بشكل دقيق وفهم وظيفته الأساسية وآلية عمله الحالية.

الفكرة الأساسية للمشروع هي أن يقوم الذكاء الاصطناعي بالإجابة على الأسئلة المطروحة عليه من خلال الـ APIs المرتبطة به. لكن في بعض الحالات، لا يتمكن من الإجابة، ويعتذر للمستخدم أو يقول إنه لا توجد إجابة، رغم أن الإجابة موجودة فعليًا داخل البيانات أو ضمن أحد الـ APIs.

## المطلوب

المطلوب منك هو تشخيص سبب الخلل بدقة:

- هل المشكلة في آلية اختيار الـ API المناسب؟
- أم في طريقة صياغة الطلب المرسل إلى الـ API؟
- أم في معالجة النتيجة القادمة من الـ API؟
- أم أن هناك ضعفًا آخر في منطق البحث أو الاسترجاع أو إعادة الصياغة؟

## مثال على المشكلة

عند سؤال النظام:

> هل لدى العتبة العباسية مشاريع زراعية؟

الإجابة الصحيحة هي أن العتبة العباسية لديها العديد من المشاريع الزراعية، لكن النظام يعتذر ويقول إنه لا توجد إجابة.

## ملاحظة مهمة جدًا

لا أريد معالجة المشكلة بإضافة ثوابت أو استثناءات أو إجابات مخصصة لحالات معينة داخل الكود، لأن هناك آلاف الأسئلة الأخرى المشابهة.

الأمثلة المذكورة هي فقط لتوضيح طبيعة الخلل وفهم المقصود، وليست حالات مطلوب تثبيتها أو برمجتها بشكل خاص.

المطلوب هو الوصول إلى حل منطقي عام يحسن آلية الفهم، واختيار الـ API المناسب، وصياغة الطلب، وتحليل النتائج، بحيث يعمل النظام بشكل صحيح مع مختلف الأسئلة وليس مع أمثلة محددة فقط.

## الآلية المفترضة

الآلية المفترضة بسيطة:

1. المستخدم يطرح سؤالًا.
2. يقوم الذكاء الاصطناعي بفهم السؤال.
3. يحدد الطلب الصحيح.
4. يرسل الطلب إلى الـ API المناسب.
5. يستقبل الجواب.
6. يعيد صياغة الجواب للمستخدم بشكل واضح ودقيق.

## الصلاحيات

لديك الصلاحية في تعديل أو حذف أو تحسين أي جزء من الآلية الحالية للوصول إلى أفضل طريقة ممكنة للإجابة، بشرط أن يتم تحسين:

- دقة اختيار الـ API.
- جودة الطلب المرسل.
- طريقة التعامل مع النتائج.
- تقليل حالات الاعتذار غير المبرر عندما تكون الإجابة موجودة.

## المطلوب النهائي

تشخيص مناطق الضعف في النظام، وتوضيح سبب فشل الإجابة في الأمثلة المشابهة، ثم اقتراح وتنفيذ التعديلات المناسبة للوصول إلى آلية إجابة أكثر دقة وموثوقية، دون الاعتماد على حلول ثابتة أو استثناءات مخصصة لأمثلة بعينها.
